# 📧 نظام البريد والإشعارات - Email & Notifications System

## نظرة عامة

تم تنفيذ نظام متكامل للبريد الإلكتروني والإشعارات يتعامل مع:
- 📌 تأكيدات الحجز
- 💬 الاستفسارات والرسائل
- 📰 اشتراكات الرسائل الإخبارية
- 🔔 إشعارات المسؤول

---

## 🏗️ البنية المعمارية

```
lib/email.ts                          ← دوال إرسال البريد الرئيسية
  ├─ sendReservationConfirmation()    ← بريد تأكيد الحجز
  ├─ sendContactAcknowledgement()     ← بريد الاستفسار
  └─ sendNewsletterWelcome()          ← بريد ترحيب النيوزلتر

app/api/reservations/route.ts         ← حفظ الحجز + إرسال البريد
app/api/contact/route.ts              ← حفظ الاستفسار + إرسال البريد
app/api/newsletter/route.ts           ← اشتراك + إرسال البريد
```

## 📋 السيناريوهات والعمليات

### 1️⃣ **عند الحجز** (Reservation)

**المسار:**
```
1. العميل يملا نموذج الحجز
                ↓
2. POST /api/reservations
                ↓
3. الحفظ في Supabase
                ↓
4. إرسال البريد:
   ✉️ تأكيد للعميل (عربي/إنجليزي)
   ✉️ إشعار للمسؤول
                ↓
5. إرجاع الرد للعميل
```

**محتوى البريد للعميل:**
- ✅ تأكيد الحجز
- 📅 التاريخ والوقت
- 👥 عدد الأشخاص
- 📞 بيانات التواصل

**محتوى البريد للمسؤول:**
- 🔔 إشعار حجز جديد
- 📋 كل تفاصيل الحجز
- 🔗 رابط مباشر لوحة التحكم

---

### 2️⃣ **عند الاستفسار** (Contact Inquiry)

**المسار:**
```
1. العميل يملا نموذج الاستفسار
                ↓
2. POST /api/contact
                ↓
3. الحفظ في Supabase
                ↓
4. إرسال البريد:
   ✉️ رسالة تأكيد للعميل
   ✉️ إشعار للمسؤول مع الرسالة
                ↓
5. إرجاع الرد للعميل
```

**محتوى البريد للعميل:**
- 👋 رسالة ترحيب
- ⏱️ وقت الرد المتوقع: 24 ساعة
- 🔗 رابط العودة للموقع

**محتوى البريد للمسؤول:**
- 🔔 استفسار جديد
- 📝 الرسالة كاملة
- 📧 بيانات المرسل
- 🔗 رابط الرد المباشر

---

### 3️⃣ **النيوزلتر** (Newsletter)

**المسار:**
```
1. العميل يدخل بريده
                ↓
2. POST /api/newsletter
                ↓
3. الحفظ في Supabase
                ↓
4. إرسال بريد ترحيب:
   ✉️ رسالة ترحيب شاملة
                ↓
5. إرجاع الرد للعميل
```

**محتوى البريد:**
- 🎉 رسالة تحية ترحيب
- 📋 قائمة بما سيتلقاها الزائر:
  - 🍋 عروض حصرية
  - 📰 أخبار جديدة
  - 🗓️ أحداث وعروض
  - 👨‍🍳 نصائح ووصفات
- ➡️ رابط الموقع

## 🚨 معالجة الأخطاء

### ما يحدث إذا فشل البريد؟

```typescript
// البريد لا يوقف العملية الرئيسية
try {
  await sendReservationConfirmation(data);
} catch (error) {
  console.error('Email failed but reservation was saved');
  // الحجز محفوظ في قاعدة البيانات رغماً عن فشل البريد
}
```

✅ **النتيجة:** العميل يتمكن من الحجز حتى لو فشل البريد  
⚠️ **ملاحظة:** يمكنك إعادة محاولة إرسال البريد من لوحة التحكم

## 🔐 الأمان

### نقاط أمان مهمة:

1. ✅ لا يتم تخزين كلمات الـ API في الـ Client

4. ✅ لا يتم حفظ البيانات الحساسة في السجلات
